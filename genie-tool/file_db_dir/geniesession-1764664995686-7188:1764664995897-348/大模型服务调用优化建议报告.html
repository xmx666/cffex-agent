2. 高延迟模型性能问题分析

glm46-fp8-local 模型在近七日中被两个高调用应用（智能客服系统、文档智能解析）持续调用，其平均延迟高达892ms与678ms，显著高于其他模型。该模型为本地部署的FP8量化版本，虽在推理速度上优于全精度模型，但其在高并发场景下存在以下问题：

  • 显存占用高，单实例并发处理能力受限，易触发GPU资源争抢
  • 缺乏动态批处理优化，请求排队严重，导致尾延迟（P95）超过2100ms
  • 本地部署环境未启用模型并行或流水线并行，无法横向扩展
  • 未配置自动扩缩容策略，高峰期资源无法动态补充

关键发现：在调用量最高的两个应用中，glm46-fp8-local 模型的延迟贡献率超过75%，是系统整体响应时间的主要拖累因素。[[2]]

此外，该模型在非峰值时段资源利用率不足30%，存在明显的资源浪费与成本效率失衡问题。

3.1 模型替换策略

针对智能客服系统与文档智能解析，建议将glm46-fp8-local替换为性能更优的云端模型：

3.2 资源扩容与负载均衡

对仍需保留glm46-fp8-local的次要应用，建议：
