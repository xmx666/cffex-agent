1. 视图模型概览

系统提供多时间粒度的行情聚合视图，用于支持不同分析需求。各视图模型均基于原始逐笔数据构建，具备标准化的时间戳与价格/成交量结构。

cffex_ohlcv_1day

时间粒度：日线

描述：每日开盘价、最高价、最低价、收盘价及成交量的聚合数据，适用于中长期趋势分析与策略回测。

格式：包含字段：timestamp（日期）、open、high、low、close、volume、open_interest

备注：数据按交易日收盘时间（15:00）进行聚合，节假日无数据。

cffex_ohlcv_1hour

时间粒度：小时线

描述：每小时开盘价、最高价、最低价、收盘价及成交量的聚合数据，适用于日内趋势识别与交易信号生成。

格式：包含字段：timestamp（日期时间）、open、high、low、close、volume、open_interest

备注：交易时段为9:30-11:30与13:00-15:00，非交易时段无数据点。

cffex_ohlcv_1minute

时间粒度：分钟线

描述：每分钟开盘价、最高价、最低价、收盘价及成交量的聚合数据，适用于高频交易策略与微观市场结构研究。

格式：包含字段：timestamp（日期时间）、open、high、low、close、volume、open_interest

备注：数据按每分钟整点聚合，非交易分钟无记录，数据延迟低于1秒。

2. 新增数据表模型：cffex_tick

作为系统底层原始数据源，cffex_tick 表存储所有逐笔成交与报价信息，是构建上述聚合视图的唯一数据基础。

cffex_tick 结构说明

timestamp DATETIME
symbol VARCHAR(20)
price DECIMAL(10,2)
volume INT
turnover DECIMAL(15,2)
bid_price DECIMAL(10,2)
bid_volume INT
ask_price DECIMAL(10,2)
ask_volume INT
trade_type ENUM('buy','sell','neutral')
sequence_id BIGINT

字段说明：

  • timestamp：成交或报价发生的确切时间，精确到毫秒。
  • symbol：合约代码，如 IF2510、IH2510 等。
  • price：成交价格，单位为人民币元。
  • volume：单笔成交数量，单位为手。
  • turnover：单笔成交金额，等于 price × volume × 合约乘数。
  • bid_price / bid_volume：买一价与买一量，反映市场买方压力。
  • ask_price / ask_volume：卖一价与卖一量，反映市场卖方压力。
  • trade_type：成交方向分类，buy（主动买入）、sell（主动卖出）、neutral（中性盘口）。
  • sequence_id：全局唯一序列号，用于确保数据顺序与完整性。

备注：该表为追加写入型表，不支持更新或删除操作，数据量级为每秒数百至数千条，需定期归档。

3. 数据模型按用途分类

行情数据（Aggregated Market Data）

由 cffex_ohlcv_1day、cffex_ohlcv_1hour、cffex_ohlcv_1minute 三个视图构成，用于宏观与中观层面的市场分析。

  • 数据经过聚合，降低噪声，提升可读性与计算效率。
  • 适用于技术指标计算（如MACD、RSI）、趋势判断、组合风险评估。
  • 数据更新频率与时间粒度一致（日/小时/分钟），非实时。

数据来源：由 cffex_tick 表按时间窗口聚合生成。

逐笔数据（Tick-by-Tick Data）

由 cffex_tick 表提供，包含每一笔成交与最优盘口的原始记录，是微观市场分析与算法交易的核心。

  • 数据粒度最细，保留完整市场微观结构信息。
  • 用于订单流分析、流动性研究、滑点建模、高频策略开发。
  • 数据为实时流式写入，延迟极低，需高性能存储与处理系统支撑。

数据用途：作为所有聚合视图的唯一原始输入源。

4. 总结与数据流关系

系统数据模型采用分层架构：底层为 cffex_tick 逐笔数据表，提供最原始、最完整的市场行为记录；上层为三个聚合视图（1day、1hour、1minute），按需对原始数据进行时间窗口聚合，以满足不同分析场景的效率与精度需求。

数据流方向为单向：cffex_tick → cffex_ohlcv_1minute → cffex_ohlcv_1hour → cffex_ohlcv_1day。每一层均依赖下一层的完整性与准确性，因此 cffex_tick 的数据质量直接决定整个系统的分析可靠性。

该模型设计兼顾了实时性、存储效率与分析灵活性，是支持量化研究与交易系统运行的核心数据基础设施。
